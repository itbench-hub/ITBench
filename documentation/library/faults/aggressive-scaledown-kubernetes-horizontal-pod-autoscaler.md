# Aggressive Scaledown Kubernetes Horizontal Pod Autoscaler

| [Source](../../../scenarios/sre/library/indexes/faults/aggressive-scaledown-kubernetes-horizontal-pod-autoscaler.json) | [Schema](../../../schemas/json/faults/aggressive-scaledown-kubernetes-horizontal-pod-autoscaler.json) | [Implementation](../../../scenarios/sre/project/roles/faults/tasks/inject_aggressive_scaledown_kubernetes_horizontal_pod_autoscaler.yaml) | Kubernetes | Deployment, Performance |

This fault misconfigures a horizontal pod autoscaler to make aggressive scale-down actions. Utilization thresholds are set to 99% so scale-up is almost never triggered. The scale-down policy is set to remove 100% of pods in 15 seconds with no stabilization window, so any brief dip in load immediately collapses the replica count. This simulates a cost-saving misconfiguration where an operator has tuned the HPA to keep as few pods as possible.

## Expectation

The faulted autoscaler will aggressively scale the workload down to its minimum replica count whenever load briefly drops, then refuse to scale back up until utilization reaches 99%. During demand surges the workload will have too few pods to handle traffic, causing latency, timeouts, and service disruption.
